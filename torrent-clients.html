Api Compatibility

By default, the api will try to use Torque, a headless torrent client. This means the connect call will facilitate the installation of a browser plugin, and the underlying torrent client if it isn't already available. This is similar to the way that browsers will handle the installation of Flash if it isn't available, freeing the Flash developer to assume that the functionality is available.

Most web apps that use btapp.js will probably want to use the headless Torque client. However, the api is also already available in the alpha branches of BitTorrent and uTorrent (version 3.3 and later), which is available via the uTorrent forums.

Browser Plugin

When calling connect for the first time, the user will be prompted to install the browser plugin. The browser plugin is responsible for making sure that the underlying torrent client is running when needed. This is usually desired when using the Torque client, however it may not be necessary if you're connecting to say, µTorrent.

If you don't wish to require the plugin, specify 

plugin: false
As the plugin also handles pairing via an embedded iframe element, it will be necessary to specify the native pairing type as below: 
pairing_type: 'native'

Pairing

Pairing is a security measure to make sure that websites you visit must be granted user approval before accessing the contents of your torrent client. This is handled by the connect call, and only needs to be specified if you don't want to use the browser plugin.

If you've specified plugin: false, use the following

pairing_type: 'native'

For reference, the default is the same as specifying the following:

pairing_type: 'iframe'

Torque

The default product that the library connects to is Torque, so there is no need to specify anything. 
var btapp = new Btapp;
btapp.connect();

this is identical to providing the default product and plugin options

var btapp = new Btapp;
btapp.connect({
  product: 'Torque',
  plugin: true
});

uTorrent

When using btapp.js to connect to a uTorrent client running on your user's machine, you may not want to 
var btapp = new Btapp;
btapp.connect({
  product: 'uTorrent',
  plugin: false,
  pairing_type: 'native'
});

BitTorrent

var btapp = new Btapp;
btapp.connect({
  product: 'BitTorrent',
  plugin: false,
  pairing_type: 'native'
});
